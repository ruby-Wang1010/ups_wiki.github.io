数据说明

原始数据说明¶

离线数据存储在S3，每天会将前一天的数据写到S3，通过分析服务器访问：

  • 访问路径：个人目录/s3_buckets/upsbucket

  • 数据构成：

- 主机：一台UPS主机可能包括1-2个电池组。
- 电池组：一个电池组内部有多个单体电池，但数据仅仅是电池组数据不包括单体电池数据。
- 单体电池：各个单体电池的数据。
  • 数据上报：

设备上报（时间间隔可能超过2s）上传到本地，采集程序从本地读取数据，读取时间间隔1s。数据上报格式是纯数据没有字段名，开发组按照顺序进行解析。

我们基于S3中开发组解析后的数据进行后续数据处理和分析，数据约为10s一条，偶尔会出现重复采集或时间跳变的情况。

  • 目录结构：
├── upsbucket                              // 数据文件夹
|   |
|   └──── 日期文件夹                       // 一个日期一个文件夹
|         |
|         └──── deviceId站点文件夹         // 一个站点一个文件夹
|               |
|               ├──── batteryBefore.csv   // 单体电池数据解析前
|               ├──── batteryAfter.csv    // 单体电池数据解析后
|               ├──── groupBefore.csv     // 电池组数据解析前
|               ├──── groupAfter.csv      // 电池组数据解析后
|               ├──── hostBefore.csv      // UPS主机数据解析前
|               └──── hostAfter.csv       // UPS主机数据解析后
  • 数据格式：

主机、电池组和单体电池解析前后的6张表中，数据的基本格式为：

时间戳，{
    "data":
        {
            数据（内容对应不同type的字段对应表）
        }
}

即以json格式存储数据，我们会将其转换为csv格式便于后续使用，处理方法见/share/UPS/stage_1/test/20200831/data_preprocess.py，处理后的数据存储在/share/UPS/data目录下。
